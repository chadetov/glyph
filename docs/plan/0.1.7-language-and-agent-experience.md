# 0.1.7 — Works with React, speaks to agents

All 15 brainstormed items are scoped into 0.1.7, built in adoption-rank order
(1 → 15). Testing is a first-class deliverable for every item: nothing lands
without tests, and a user-visible feature also updates the website Q&A (per the
CLAUDE.md convention) and the roadmap.

Standing test bar for every item:
- **Rust unit/integration tests** in the crate that changed (parser snapshots,
  emit assertions, typechecker cases).
- **A negative case** in `tests/negative/` when the item adds or changes a
  diagnostic (a program that must fail with a named code).
- **End-to-end** where it produces runnable output: `glyph build --check`
  (tsc-clean) and, where it runs, `glyph run`.
- **No regressions**: the full `cargo test --workspace` stays green, and the
  examples/corpus still `tsc --strict` clean.

## 1. JSX fragments `<>...</>` (S/M)

Parser + emitter. `<>...</>` lowers to `React.Fragment` (or the array/`<>`
form the emitter already uses for children).
- **Parser**: recognize `<>` / `</>` as a fragment element with children and no
  tag/attrs. Reuse the existing JSX child parsing.
- **Emitter**: emit `React.createElement(React.Fragment, null, ...children)`.
- **Testing**: parser snapshot for a fragment with mixed children; emit test
  asserting the `React.Fragment` lowering; an example/corpus program using a
  fragment that `tsc --strict` accepts.

## 2. Member-expression JSX `<Ns.Comp>` (S/M)

Parser + emitter. A JSX element name may be a dotted path (`Ctx.Provider`).
- **Parser**: `jsx_name` / element-name parsing accepts `A.B.C`.
- **Emitter**: emit the member expression as the `createElement` type
  (`React.createElement(Ctx.Provider, ...)`).
- **Testing**: parser snapshot; emit test that `<Ctx.Provider value={x}>`
  becomes `createElement(Ctx.Provider, { value: x }, ...)`; a Context example
  that `tsc --strict` accepts.

## 3. Machine-readable diagnostics (`--json`) (M)

A `--json` flag on `glyph build` (and `run`) that emits diagnostics as a JSON
array instead of ariadne text: `{ code, severity, message, file, span:{start,
end, line, col}, help, note }`. The wedge feature for agents consuming Glyph.
- **Approach**: a `Diagnostic` value type carried through the build (today
  diagnostics are pre-rendered strings; add a structured form alongside).
  Serialize with serde. Include remapped `tsc` diagnostics too.
- **Testing**: unit tests that a known program produces the expected JSON
  (stable `code`/`span`); a schema-shape test; `--json` on a clean build emits
  `[]`.

## 4. Runtime source maps (M/L)

Emit a standard `//# sourceMappingURL` + `.map` (VLQ) so a debugger and stack
traces step into `.glyph`. Builds on the compile-time source map from 0.1.6.
- **Approach**: extend the emitter's checkpoint map to column granularity where
  cheap, generate a v3 source map per module, write the `.map` and the comment.
- **Testing**: assert a valid v3 map is written (parse it back, spot-check a
  mapping); an end-to-end where a thrown error's stack references `.glyph`.
- **Risk/scope**: if full VLQ mapping is heavy, ship line-level mappings first.

## 5. `gen dts` on TypeScript 7 native API (M/L)

Drive the `typescript/unstable/*` project/handle API so the default
`npm install typescript` (7.x) works, not just a pinned `typescript@6`.
- **Approach**: extend `ts-to-schema.mjs` to detect 7.x and use the
  `unstable/sync` `API` → project → program → source-file path (scouted in
  0.1.5); fall back to the classic API when present.
- **Testing**: the existing `gen dts` test runs under both 6.x and 7.x when
  available; a fixture `.d.ts` maps identically regardless of compiler version.

## 6. Bounded generics `<T: Bound>` (M)

Parser + checker + emit. `fn f<T: Show>(x: T)` constrains `T`.
- **Parser**: generic param accepts an optional `: Bound`.
- **Checker**: record the bound; verify a type argument satisfies it (v1 scope:
  a named bound the arg's type must name/implement — keep conservative, no full
  trait system).
- **Emit**: TS `<T extends Bound>`.
- **Testing**: parse snapshot; emit `<T extends Bound>`; a negative case where
  an unsatisfied bound errors; tsc-clean positive.

## 7. Discriminated-union generation (L)

Give a Glyph union a runtime representation that discriminates on a *named
property with a string value* (OpenAPI `discriminator`), not the built-in
`{tag: "Cat"}`. Then map a discriminated `oneOf` faithfully.
- **Approach**: a descriptor/representation option so `gen openapi`/`gen zod`
  can emit a union whose `.parse` reads the discriminator property. This is a
  language/runtime capability, not a mapper tweak — scope carefully; a
  documented subset if the general case is too large.
- **Testing**: generated union `.parse` accepts the real wire object
  (`{petType:"cat", ...}`) and rejects mismatches; round-trip idempotent.

## 8. Shared-state / store pattern (M, design first)

Today shared mutable state has one legal home (a `let` in `main`). Design a
clean store module pattern (no new linear types, no relaxed `mut` rules).
- **Approach**: write the design note first (which pillar it serves, why it
  doesn't relax `mut`), then implement — likely a stdlib `store` helper, not a
  language change.
- **Testing**: a worked example app using the store; the pattern `tsc`-clean and
  runnable.

## 9. More warning-tier lints (S each)

On the 0.1.6 warning tier: unused import, unused `let` binding, unreachable
code after a diverging statement. Each a new warning code.
- **Testing**: per lint, a positive (warns, still emits) and a clean case (no
  false positive across examples/corpus). Follow the E0217 pattern.

## 10. number/string value-match exhaustiveness (M)

Extend match exhaustiveness to number/string literal scrutinees (a match on
values must be total or have an `else`).
- **Testing**: a negative case (non-exhaustive value match → a coded error); a
  positive with `else`; no regression on existing bool/union exhaustiveness.

## 11. `glyph regen` (M)

Implement the `@generate`-spec → regenerate-body command (Q40), consistent with
the `gen` family.
- **Testing**: a fixture with a `@generate` block regenerates deterministically;
  idempotent re-run.

## 12. `@redact` full enforcement (D24) (M)

Real masking in the emitted serializer/descriptor + field-name validation. A
prior autonomous attempt broke the descriptor tests — retry carefully.
- **Testing**: a redacted field is masked in serialized output; the descriptor
  tests stay green; a negative for an unknown redacted field name.

## 13. `--out` cleans stale files first (S)

`glyph build --out X` removes stale emitted files before writing (a
renamed/deleted module leaves no orphan `.ts`). `run` already cleans; make
`build` consistent without clobbering unrelated files.
- **Testing**: build, rename a module, rebuild — the old `.ts` is gone; a
  non-emitted file the user placed is preserved (or the policy is documented).

## 14. Extend the type-name hint to `int`/`any`/`Promise<T>` (S)

The `bool`-style "did you mean the Glyph spelling" hint, extended.
- **Testing**: writing `int`/`any` yields the targeted hint; snapshot.

## 15. Nested nullary-in-object parser bug (S)

`Wrap({ inner: Inner })` as a record-payload variant in first position doesn't
parse. Fix the parser; it should accept the declaration.
- **Testing**: a parse snapshot for the previously-failing declaration; the
  emitted match lowers and is tsc-clean.
