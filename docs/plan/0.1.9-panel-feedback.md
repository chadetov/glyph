# Glyph 0.1.9 — Panel-Feedback Release Plan

## Release thesis

0.1.9 is the **trust release**: it closes the first-touch credibility gap (an empty npm page, a looping "tour" link, an undefined "agents," a redirect-only `/answers/`), answers the one decisive objection every serious evaluator holds ("why not strict TS + zod + eslint + ts-pattern?") with honest content and the *apparatus* for a real differential agent study, and fixes the concrete correctness/UX defects the panel actually hit (raw Node stack traces from `glyph run`, a temp-dir race, always-empty `issue.path`, descriptors that ignore unknown keys and silently narrow fields they can't verify). The through-line is **honesty as the conversion strategy**: every claim on the site and in `llms.txt` is either backed by a shipped code path / test or explicitly labeled a current edge — because the persona study concluded that naming the gaps is what earned trust, not hiding them.

**What the 12 personas told us.** The unanimous #1 blocker was discovery friction that reads as *abandoned*: "No README data found" on npm, a tour link that loops home, a personal-account repo with no contribution on-ramp. The decisive *content* gap was that the site argues Glyph vs Go / new runtimes but never confronts tooled TypeScript, and the current `2 of 2` / `n=3` benchmark framing actively hurts. The decisive *product-honesty* gaps were runtime-validation overclaims (`issue.path` always empty, extra keys accepted, `@redact`/`owned` marketed past what ships) and a handful of sharp edges (main-less `glyph run`, unpinned toolchain, unsigned artifacts). Nobody who reached an honest answer walked away — the fix is measured small claims with the edges named, not bigger unmeasured ones.

## Testing strategy for 0.1.9

The standing bar, applied per task:

1. **Rust unit + integration.** New helpers get unit tests (`module_exports_main`, `remove_dir_all_retry`, `descriptor_verifiability`, doctor version-parse, `score.py`/`judge.py` fixtures). New behaviors get integration tests: C1 exit-code-2 + no-`TypeError`; C2 the 8-thread race regression; C3a nested/array `path`; C4 doctor with a scrubbed `PATH`; C5 scaffold JSON assertions; B9 the `--json` envelope pin.
2. **A negative case per new diagnostic.** `tests/negative/` gains fixtures for `@open` misplacement (C3b) and, where the code path is a compile failure, each new code. E0220 (a warning) instead gets a positive-with-warning CLI assertion (build succeeds, exact code surfaced, warned-set == emitter fallback-set). E0310 is a run-time usage error → an integration assertion (exit 2), not a `negative/` fixture.
3. **`tsc --strict` on examples + corpus.** The existing CI gate must stay green through the C3 descriptor rewrite; B3 extends the glob to `examples/interop/*`; every Glyph snippet shown on a new answer page is extracted and built.
4. **Site HTML well-formedness + consistency.** D6 harness: internal link-check across `web/**`, both READMEs, `docs/**`; the answers subnav/dir/eyebrow/prevnext/sitemap consistency script (D5); the changelog-vs-registry check (A4); the `npm pack` README-presence assertion (A1); light/dark render + no-horizontal-scroll spot-checks for the new tables (D4). Every published site number carries a traceable `data-source` (B3/B10). Error codes cited on pages must exist in `docs/error-codes.md` (grep gate).
5. **No regressions.** `cargo test --workspace` green (currently 380+); `bootstrap.rs` mirror + content guards green after every `AGENTS.md` edit (author in `AGENTS.md`, then re-copy to `llms.txt` + `web/llms.txt`); benchmark `run.py --replay --verify` reproduces every committed `results/*.json` offline.

**Honesty regression guards (the release's signature):** snapshot the real `E0006`/`E0200` diagnostic text the tutorial and home page display and diff against live output; a negative fixture asserting extra keys are *not* rejected **until C3b lands** (so the pages can't silently become a lie); the benchmark task-01 hidden case documents the extra-key gap's current state and flips when C3b closes it.

## Release shape & sequencing

**Honest capacity call.** All of the *docs/site/agent-surface/supply-chain* work and the *compiler/UX fixes* fit one 0.1.9 — they are overwhelmingly S/M and mostly independent. The *benchmark* fits **as apparatus + a small real pilot only** (E1); the full study is 0.2.x. The *big language items* do **not** fit as builds — 0.1.9 ships all six as S honesty fixes plus, if capacity allows, **at most E2a (`?` widening) and/or E2b (`owned` handle)**, both M and self-contained. `decimal`, generic/union descriptors, the trait system, and automatic `@redact` are staged to 0.2.x/0.2.0 with written sketches. This is the realistic shape: 0.1.9 is *many small honest fixes + the decisive content + the study apparatus*, not the heavy language builds.

**Build order (what unblocks what):**

1. **Foundations first (parallel):** D6 site-validation harness (guards every later doc edit); the **shared `descriptor_verifiability` predicate** + the C3 descriptor pass plan (C3a/b/c touch the same emit functions — do them as one coordinated pass, not three); the LANG honesty fixes A10 (pure text, unblock nothing, ship anytime).
2. **Compiler track (parallel with docs):** C1, C2, C4, C5 are orthogonal CLI work; C3 is the one coordinated pass. E2a/E2b slot here if pulled in. New codes reserved: **E0310** (run, no main), **E0220** (descriptor partial-check warning).
3. **Supply-chain track (fully parallel):** A6/A7/A8 land first (all in `release.yml`, no deps) → A9 docs (needs final filenames) → D7/D8 VS Code packaging (independent) → D9 publish wiring (human-gated).
4. **Content track:** A1–A5 (front-door S wins) → B7/B8/B9 + D1 (AGT bootstrap, author in `AGENTS.md`) → B1/B2/B3 + D4 (positioning pages + CSS) → B4/B5/B6 (tutorial + home + human-value) → E1 + B10/B11 (benchmark apparatus + relabel + reliability page).
5. **The chokepoint, late:** **D5 (answers renumber)** runs *once*, only after the final answer set is frozen (all new pages from POS/BENCH/FD decided). D3 (`/answers/` index) follows D5. D11 (`llms.txt` pointers) after the pages exist. D10 (org note) anytime; D2 (CONTRIBUTING) anytime.
