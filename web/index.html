glyph

Code an agent can change without breaking it.

A statically typed language that transpiles to TypeScript, designed so AI agents can read, write, and modify code safely.

Glyph looks almost like TypeScript — a TS developer reads it on day one without a tutorial. The differences are deliberate and few, and each one exists to make code an agent can reason about correctly, edit without breakage, and explain back to a human without lying.

The four pillars

Verifiability

What the type system claims is true at runtime. No any, no structural-typing surprises, no type erasure.

Greppability

Every symbol has one syntactic form at its declaration site. grep "fn parseUser" finds the definition. Always.

Abstraction

Express intent at the level you are thinking: pattern matching over switch ladders, Result over thrown exceptions.

Diff stability

A one-line change produces a one-line diff. Fixed formatting, one element per line, trailing commas everywhere.

Why Glyph

  • Compiles to TypeScript. Runs anywhere TS runs; uses any npm package.
  • Looks like TypeScript. Readable on day one, no tutorial.
  • No any. What the types say is true when the code runs.
  • Errors are values. You handle them with match, not exceptions.
  • match covers every case. The compiler tells you what you missed.
  • Tests live next to the code. They run on every build.
  • One name, one form. grep always finds a definition.
  • Clear errors. Each one tells you how to fix the problem.
