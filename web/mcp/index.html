For agents · Model Context Protocol

Wire your agent into Glyph's compiler

glyph mcp runs a Model Context Protocol server that hands a coding agent Glyph's own understanding of your project as tools — diagnostics, the type at a cursor, where a name is defined, every reference to a symbol, symbol search. It's the same analysis the editor uses, exposed over stdio, so what the agent sees can't drift from what the compiler sees.

Why an agent wants this

An agent editing a codebase usually works blind: it reads files, guesses at types, and finds out whether it broke something only after a build. Glyph already answers those questions precisely for an editor. glyph mcp gives an agent the same answers directly — ground truth from the compiler, not inference — without scraping build output or reading every file to find who calls what. It complements glyph build --json (the batch path for coded diagnostics) with interactive, position-level queries.

Set it up

Any MCP client can launch it. Point the client at the glyph binary with the mcp subcommand; pass your project root as an argument, or run it from the project directory. It speaks MCP over stdio — no port, no daemon.

mcp client config (generic) 
{
  "mcpServers": {
    "glyph": {
      "command": "glyph",
      "args": ["mcp", "/path/to/your/project"]
    }
  }
}

On first connect the client calls initialize and tools/list; the server reports the five tools below. Positions are the LSP convention — a 0-based line and a 0-based character (UTF-16 code units).

The five tools

tools exposed by glyph mcp 
// type-check one file → diagnostics with stable codes + ranges
glyph_diagnostics(path)

// the inferred type at a position
glyph_hover(path, line, character)

// where the name is defined, following imports across modules
glyph_definition(path, line, character)

// every reference across the whole project: the declaration, all
// uses, and each importing module's `import` binding
glyph_references(path, line, character)

// search top-level declarations and union variants by name
glyph_symbols(query)

See it

A tools/call for glyph_diagnostics on a file with a field typo returns the compiler's own diagnostic — the stable code, the message, and the exact range — as structured JSON the agent parses and acts on:

request 
{
  "method": "tools/call",
  "params": {
    "name": "glyph_diagnostics",
    "arguments": { "path": "src/user.glyph" }
  }
}
result (the tool's JSON content) 
[
  {
    "code": "E0210",
    "message": "type `U` has no field `naem`",
    "range": {
      "start": { "line": 7, "character": 9 },
      "end":   { "line": 7, "character": 15 }
    }
  }
]

glyph_references is workspace-wide: ask about a symbol and you get its declaration, every use, and each importing module's import binding, across every file — the same cross-file identity the editor's rename uses, so the answer is complete, not just same-file.

Honest scope

The tools are read-only queries today — the agent reads Glyph's understanding; it doesn't apply edits through MCP (a rename tool that returns edits is a planned follow-up). The workspace is scanned per request rather than cached (fine for real projects, a speed optimization later), and a file that doesn't currently parse is skipped. A local binding's references are file-scoped, since a local can't be named from another file. And because every tool is a thin adapter over the same pure analysis layer the editor calls, none of it is a second implementation that could disagree with the compiler.

Where it stands

Shipping today

An glyph mcp [root] server over stdio with five tools — glyph_diagnostics, glyph_hover, glyph_definition, glyph_references (workspace-wide), and glyph_symbols — each a thin adapter over the same analysis the language server uses.

On the way

A rename tool that returns the edits to apply, a cached/incremental workspace index, and extending cross-file resolution to more of the query surface.

Not using MCP? An agent can still get the full language and diagnostic surface offline via llms.txt (the spec) and glyph build --json (coded diagnostics). MCP is the interactive layer on top.
